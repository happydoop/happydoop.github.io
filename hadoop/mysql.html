¶ MySQL

本节以Mysql 8 为例进行讲解。 由于最新版的 CentOS 7 默认安装的是 mariadb 。我们需要先卸载它，不然可能会对 Mysql 的安装造成影响。

首先搜索 mariadb

rpm -qa | grep mariadb
1

可以看到

mariadb-libs-5.5.65-1.el7.x86_64
1

将出现的所有 mariadb 相关都删除，如

rpm -e --nodeps mariadb-libs-5.5.65-1.el7.x86_64
1

¶ 安装

首先下载。这里使用的是 mysql-8.0.12 (opens new window)。

解压：

xz -d mysql-8.0.12-linux-glibc2.12-x86_64.tar.xz
tar -xf mysql-8.0.12-linux-glibc2.12-x86_64.tar
1
2

之后将其移动到 /usr/local/mysql 目录下：

mv mysql-8.0.12-linux-glibc2.12-x86_64 /usr/local/mysql
1

在使用 mysql 时，最好先创建一个 mysql 用户：

useradd mysql # 创建 mysql 用户
passwd mysql # 设置 mysql 用户的密码
1
2

之后，进入 mysql 目录，将其所有者更改为 mysql 用户：

cd /usr/local/mysql
chown -R mysql:mysql ./
1
2

然后使用 root 权限配置环境变量，vim /etc/profile：

MYSQL_HOME=/usr/local/mysql
PATH=$PATH:$JAVA_HOME/bin:$MYSQL_HOME/bin
export PATH JAVA_HOME CLASSPATH
1
2
3

记得使更改生效：

source /etc/profile
1

再修改 mysql 的配置文件（如无，则创建），vim /etc/my.cnf：

[mysqld]                                                                                        
basedir=/usr/local/mysql                                                                        
datadir=/usr/local/mysql/data                                                                   
port=3306                                                                                       
socket=/tmp/mysql.sock                                                                          
pid-file=/usr/local/mysql/data/mysql.pid                                                        
sql_mode=NO_ENGINE_SUBSTITUTION,STRICT_TRANS_TABLES
1
2
3
4
5
6
7

接下来，初始化 mysql：

mysqld  --initialize  --user=mysql  --basedir=/usr/local/mysql  --datadir=/usr/local/mysql/data
1

终端会输出类似于这样的日志：

2020-11-05T09:21:53.548925Z 0 [System] [MY-013169] [Server] /usr/local/mysql/bin/mysqld (mysqld 8.0.12) initializing of server in progress as process 11617
2020-11-05T09:21:55.000112Z 5 [Note] [MY-010454] [Server] A temporary password is generated for root@localhost: tLS9Omfj<Ne8
2020-11-05T09:21:56.207276Z 0 [System] [MY-013170] [Server] /usr/local/mysql/bin/mysqld (mysqld 8.0.12) initializing of server has completed
1
2
3

其中tLS9Omfj<Ne8是默认的初始密码。你的默认初始密码可能跟这里不一样。

然后，将 mysql 添加到启动服务中，这样以后就能使用 systemctl start mysqld 或者 service mysqld start 启动了：

cp /usr/local/mysql/support-files/mysql.server   /etc/init.d/mysqld
chkconfig  --add mysqld
1
2

最后，启动 mysql 服务：

systemctl start mysqld
1

此时，mysql 的服务已启动。

接下来，我们进入 mysql 的命令终端：

mysql -uroot -p
1

然后，输入刚才生成的默认初始密码 tLS9Omfj<Ne8，就会进入 mysql 客户端了：

到这里还没结束，我们现在的密码是默认的密码，不能正常使用，需要更改密码：

alter user  'root'@'localhost' identified by 'root';
1

上面这句命令，将 mysql 中的 root 用户的密码改为了 root（你也可以设为自己喜欢的密码，确保不会忘记）。

然后输入 exit; 退出，再重新登录：

mysql -uroot -p
1

输入密码 root，重新进入 mysql 客户端，输入 show databases; 查看已有的数据库：

为了方便之后我们能远程访问数据库，在 mysql 客户端中分别执行以下命令：

use mysql;
update user set host = '%' where user ='root';
flush privileges;
1
2
3

¶ 创建数据库

语法：

CREATE DATABASE 数据库名;
1

如，

CREATE DATABASE hadoop_test;
1

删除数据库使用 DROP DATABASE 命令，如

DROP DATABASE hadoop_test;
1

¶ 选择数据库

在进一步对数据库中的表进行操作之前，需要先选择数据库：

use hadoop_test;
1

¶ 创建表

CREATE TABLE `test_table` (
  `id` INT UNSIGNED AUTO_INCREMENT,
  `username` varchar(255) NOT NULL,
  `gender` varchar(255) DEFAULT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8;

详细讲解其中每一个参数超过了本文的范畴，可参与MySQL 创建数据表 (opens new window)。

删除表使用：

DROP TABLE test_table;
1

¶ 插入数据

语法：

INSERT INTO table_name ( column_1, column_2,... ) VALUES ( value1, value2,... );
1

如

INSERT INTO `test_table` (id, username, gender) VALUES (1, 'zhangsan', 'male');
1

¶ 查询数据

语法：

SELECT column_1,column_2
FROM table_name
[WHERE 条件语句]
1
2
3

如：

SELECT username, gender WHERE id = 1;
1

¶ 更新数据

语法：

UPDATE table_name SET column_1=new-value1, column_2=new-value2 [WHERE 条件语句]
1

如

UPDATE test_table SET username='liming' WHERE id=1;
1

不要忘了 where 语句，否则，所有记录的 username 都将被更新。

¶ 删除数据

语法：

DELETE FROM table_name [WHERE 条件语句]
1

如

DELETE FROM test_table WHERE id=1;
1
